HTML,CSS and the DOM

If a webpage were an American presidential inaugural address, the html would be the words and structure of a speech, the CSS would be how the voice was used, some words shouted or whispered for impact. Perhaps the speech is said in a different accent. The DOM would perhaps be the script or blueprint of the speech written on flash cards, where you can move different sections of the speech around. Maybe start with God bless America, maybe end with it. This was not a great analogy.

Boxifying design

This is looking at the elements on a webpage as a series of boxes organised on the page. Design is all about how to use these boxes, shaping them through CSS and organising them on the page.

The box model

Each element or "box" on a webpage consists of a margin, border, padding and content that can be manipulated to create positive and negative space on a webpage
